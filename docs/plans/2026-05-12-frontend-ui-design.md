# 资产分析 BI 平台前端 UI/UX 设计方案

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 为资产分析 BI 平台定义一套可直接落地的前端页面结构、视觉方向、组件规范和页面功能布局，覆盖登录页、全局框架、领导 Dashboard 与管理后台。

**Architecture:** 前端采用 `Next.js + Tailwind CSS + shadcn/ui + ECharts + TanStack Table`。设计上采用统一的深色专业仪表盘风格，将“领导查看”与“管理员操作”放入同一套设计系统中，通过布局层级、信息密度和视觉权重进行区分。

**Tech Stack:** `Next.js`、`TypeScript`、`Tailwind CSS`、`shadcn/ui`、`ECharts`、`TanStack Table`、`TanStack Query`、`Zustand`

---

## 1. 设计目标

本项目的前端设计需要同时满足两类核心场景：

- 面向领导层的高质量可视化展示
- 面向管理员的高效率数据管理和操作

因此整体 UI 设计必须兼顾以下目标：

- 具备接近 Grafana 的专业仪表盘气质
- 保持后台系统的可读性、稳定性和操作效率
- 统一登录页、导航、筛选器、图表卡片、表格和表单风格
- 支持后续逐步扩展为更完整的 BI 平台

## 3. 视觉方向

### 3.1 视觉风格

整体风格建议采用：

- 深色专业仪表盘风格
- 高级中性色背景
- 低饱和蓝紫高亮色
- 半透明卡片与细边框
- 强调图表、指标和状态色的层级感

目标感觉不是传统 ERP，也不是炫技大屏，而是：

- 有管理系统的秩序感
- 有分析平台的专业感
- 有面向领导汇报的视觉完成度

### 3.2 色彩建议

建议的主色板如下：

- 页面主背景：`#0b1020`
- 内容区背景：`#111827`
- 卡片背景：`rgba(17, 24, 39, 0.78)`
- 一级边框：`rgba(148, 163, 184, 0.16)`
- 一级文字：`#f8fafc`
- 二级文字：`#94a3b8`
- 主色：`#60a5fa`
- 辅色：`#8b5cf6`
- 成功：`#22c55e`
- 警告：`#f59e0b`
- 危险：`#ef4444`
- 信息强调：`#38bdf8`

### 3.3 字体建议

优先推荐：

- 英文与数字：`Montserrat`
- 中文：`PingFang SC` / `Noto Sans SC`

备选组合：

- `Inter + Noto Sans SC`
- `Montserrat + Source Han Sans SC`

### 3.4 圆角与阴影

- 页面大卡片圆角：`20px`
- 标准卡片圆角：`16px`
- 输入框/按钮圆角：`12px`
- 小标签圆角：`999px`

阴影建议克制，避免过重：

- 卡片阴影：`0 8px 30px rgba(0, 0, 0, 0.24)`
- 弹层阴影：`0 12px 40px rgba(0, 0, 0, 0.32)`

## 4. 设计系统基础 Token

### 4.1 间距系统

- `4`：微间距
- `8`：组件内最小间距
- `12`：标签、表单项间距
- `16`：标准间距
- `20`：卡片内常用间距
- `24`：模块内大间距
- `32`：区块级间距

### 4.2 字号系统

- `12px`：辅助说明
- `13px`：表格说明文字
- `14px`：正文、表单
- `16px`：卡片正文、导航
- `18px`：区域小标题
- `24px`：页面标题
- `32px`：大数字 KPI

### 4.3 组件状态

所有核心组件必须具备：

- 默认态
- Hover 态
- Active 态
- Focus 态
- Disabled 态
- Loading 态
- Empty 态
- Error 态

### 4.4 无障碍与可用性

- 文本与背景对比度尽量达到 WCAG AA
- 所有按钮和输入框保留明显 focus ring
- 表格支持键盘聚焦
- 图表区域提供标题、时间范围、筛选说明
- 错误提示尽量具体，不只显示“失败”

## 5. 全局框架设计

## 5.1 页面骨架

全局页面建议采用三层结构：

- 左侧 Sidebar：一级导航入口
- 顶部 Header：全局搜索、时间筛选、通知、用户信息
- 中央 Workspace：页面主体内容区

### 5.2 Sidebar 设计

建议宽度：

- 展开态：`260px`
- 折叠态：`84px`

Sidebar 内容：

- 顶部 Logo 区
- 中部导航菜单
- 底部主题切换和当前用户组织信息

菜单项样式建议：

- 默认态低对比文字
- Hover 为浅色高亮块
- 激活态采用主色渐变边框或柔和蓝紫高亮底

### 5.3 Header 设计

Header 右侧建议包含：

- 当前页面面包屑
- 全局搜索入口
- 时间范围筛选
- 快捷导入按钮
- 通知中心
- 用户头像菜单

### 5.4 Workspace 设计

内容区宽度以流式布局为主，保留最大宽度限制：

- 常规页面最大宽度：`1600px`
- Dashboard 页面允许更宽

页面内统一结构：

- 页面标题区
- 筛选操作区
- 主内容区
- 右上角快捷操作

## 6. 登录页设计

登录页不是简单表单页，建议作为品牌建立入口。

### 6.1 布局结构

采用左右分栏布局：

- 左侧为品牌展示区
- 右侧为登录表单区

### 6.2 左侧内容

- 产品名称：资产分析 BI 平台
- 副标题：资产、采购、发货、汇总分析一体化
- 3 个能力卖点卡片
- 背景可采用深色渐变 + 网格线 + 数据图形装饰

### 6.3 右侧内容

- Logo
- 欢迎标题
- 账号输入框
- 密码输入框
- 记住登录
- 登录按钮
- 忘记密码
- 登录错误提示区域

### 6.4 交互要求

- 登录按钮必须支持 loading 状态
- 表单错误应定位到字段级别
- 登录失败给出明确提示
- 支持回车提交

## 7. Dashboard 页面设计

Dashboard 是整个平台最重要的视觉页面，面向领导查看，信息应重点突出。

### 7.1 页面结构

推荐采用以下布局：

- 第一行：页面标题 + 时间筛选 + 导出按钮
- 第二行：核心 KPI 卡片 4 到 6 个
- 第三行：年度采购趋势 + 发货趋势
- 第四行：资产类型占比 + 地点分布 + 供应商采购排行
- 第五行：关键明细表，例如最近采购记录或发货流水

### 7.2 KPI 卡片

推荐 KPI：

- 资产总量
- 当年采购总数
- 当年采购总金额
- 发货总量
- 在库资产数
- 异常资产数

卡片结构建议：

- 左上标题
- 中间大数字
- 下方同比/环比趋势
- 右上角图标或迷你趋势线

### 7.3 图表卡片规则

每张图表卡片统一包含：

- 标题
- 指标说明
- 时间范围
- 筛选器联动说明
- 图表主体
- 空状态/异常状态

### 7.4 Dashboard 页面功能

- 全局筛选器联动所有图表
- 点击图表柱子、折线点、饼图块后可跳转对应明细页
- 支持导出截图或 PDF
- 支持保存领导常用视图

## 11. 导入中心设计

导入中心是本项目的重要差异化页面，必须做得清晰、可追溯。

### 11.1 页面结构

- 上传入口区
- 最近导入任务列表
- 导入状态卡片
- 字段识别结果区
- 错误与警告区

### 11.2 页面功能

- 上传 Excel
- 选择 Sheet
- 自动识别列
- 自动推断字段类型
- 显示识别置信度
- 允许用户修正映射
- 支持原样导入
- 支持保存导入模板

### 11.3 导入流程 UI

建议采用步骤条：

- 第一步：上传文件
- 第二步：识别表结构
- 第三步：确认字段映射
- 第四步：校验数据
- 第五步：导入完成

### 11.4 状态设计

- 成功：绿色状态条和成功摘要
- 警告：黄色状态条，提示哪些字段未标准化
- 失败：红色错误区，支持下载失败明细

## 12. 字段映射与模板页设计

该页面是动态字段能力的核心后台页面。

### 12.1 页面功能

- 查看系统标准字段
- 查看导入识别字段
- 建立字段映射关系
- 调整字段类型
- 配置字段是否可筛选、可展示、可统计
- 将字段保存为模板

### 12.2 页面布局

- 左侧：Excel 原始字段列表
- 中间：映射关系编辑器
- 右侧：系统标准字段与字段属性面板

## 16. 页面状态设计

每个核心页面都必须设计以下状态：

- 首次加载骨架屏
- 空数据状态
- 数据加载失败状态
- 权限不足状态
- 无筛选结果状态

示例要求：

- Dashboard 空状态不只显示“暂无数据”，而要提示用户先导入资产或采购数据
- 导入中心失败状态应清晰展示行号、列名、失败原因
- 表格无结果状态要保留“清空筛选”操作
